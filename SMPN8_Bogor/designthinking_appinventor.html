BAGIAN 2

Design Thinking yang Mudah Dipahami

Apa itu Design Thinking?

Design Thinking adalah cara berpikir untuk mencari solusi yang berawal dari kebutuhan siswa.

Empati
Definisikan Masalah
Cari Ide
Membuat Prototipe

Problem space
Solution space
*Bukan merupakan proses linear

Kita mencoba memahami apa yang mereka rasakan, pikirkan, dan butuhkan.

Kita bekerja sama, banyak mencoba, dan tidak takut salah.

Prinsipnya:

Solusi yang dihasilkan harus membantu siswa belajar lebih baik.

Problem Space

Empati

Pada tahap ini, kita berusaha mengenal siswa lebih dekat. Apa yang mereka inginkan? Apa yang mereka butuhkan? Apa tujuan mereka?

Kita tidak menebak-nebak berdasarkan asumsi, tetapi menggali fakta dari kelas.

Hal yang bisa dilakukan:

  • Mengajak ngobrol siswa
  • Mengamati perilaku mereka di kelas
  • Empathy map

Empathy Map

Apa yang siswa KATAKAN: Berisi ucapan langsung dari siswa
Apa yang siswa PIKIRKAN: Apa yang mungkin mereka pikirkan tapi tidak selalu diungkapkan
Apa yang siswa LAKUKAN: Perilaku nyata yang kita lihat
Apa yang siswa RASAKAN: Emosi yang muncul dalam proses belajar

Solution Space

Cari Ide

Aturan Brainstorming:

  1. Jangan terburu-buru menilai ide orang lain
  2. Ajak munculnya ide-ide liar dan kreatif
  3. Kembangkan ide teman ‚Äî bukan bersaing

  6. Gunakan gambar/sketsa bila perlu

Langkah 2: Desain Halaman Materi

  1. 1. Klik Add Screen -> Beri nama: ScreenMateri.
  2. 2. Hilangkan TitleVisible.
  3. 3. Set BackgroundImage dengan gambar materi pelajaran.
  4. 4. Buat Tombol Kembali:
    • Tarik HorizontalArrangement (Height 80%, BG None).
    • Tarik Button (btn_back) di bawahnya.
    • Ganti Image dengan ikon panah & Hapus Text.
[SCREENSHOT: ScreenMateri Full Gambar + Tombol Back Kecil]

Langkah 3: Desain Halaman Kuis

  1. 1. Add Screen: ScreenLatihan.
  2. 2. Masukkan Label (lbl_soal) -> FontSize 20, Bold, Text Center, Warna Putih.
  3. 3. Masukkan 4 Button Pilihan Jawaban:
    • Rename: btn_opsiA, btn_opsiB, btn_opsiC, btn_opsiD.
    • Set Width: Fill Parent.
    • Set Shape: Rounded.
[SCREENSHOT: Tampilan Soal & 4 Tombol Pilihan]

Langkah 4: Desain Halaman Skor

  1. 1. Add Screen: ScreenSkor.
  2. 2. Beri Background (misal gambar piala).
  3. 3. Tambahkan Label (lbl_skor_akhir).
  4. 4. Atur FontSize 40, Bold, Text "0", dan posisikan di tengah.
[SCREENSHOT: Halaman Skor Sederhana]

Langkah 5: Coding Navigasi

Screen1 (Menu)

when btn_materi.Click
open another screen "ScreenMateri"
when btn_latihan.Click
open another screen "ScreenLatihan"

ScreenMateri (Back)

when btn_back.Click
open another screen "Screen1"

Langkah 6: Coding Data Kuis

Di ScreenLatihan, buat variabel (Variables -> Initialize global):

soal: make a list ("Soal 1", "Soal 2")
kunci_jawaban: make a list ("A", "B")
opsiA: make a list ("Jawaban A1", "Jawaban A2")
skor: 0
index_soal: 1
[SCREENSHOT: Blok Variabel Lengkap]

Langkah 7: Logika Tampil Soal

Buat Procedure "TampilSoal":

to procedure TampilSoal do
set lbl_soal.Text to select list item (list: get global soal, index: get global index_soal)
set btn_opsiA.Text to select list item (list: get global opsiA, index: get global index_soal)
... (Lakukan juga untuk Opsi B, C, D) ...

PENTING: Panggil "Call TampilSoal" di blok "When ScreenLatihan.Initialize".

Langkah 8: Cek Jawaban

Logika saat tombol ditekan (Contoh: btn_opsiA):

when btn_opsiA.Click
IF (btn_opsiA.Text == select list item (kunci_jawaban, index_soal))
THEN set global skor = get global skor + 50
set global index_soal = get global index_soal + 1
IF (index_soal > length of list(soal))
THEN open another screen with start value ("ScreenSkor", get global skor)
ELSE call TampilSoal

Copy-Paste blok ini untuk btn_opsiB, C, dan D.

Langkah 9: Skor & Build

ScreenSkor

when ScreenSkor.Initialize
set lbl_skor.Text to get start value

Build APK

  1. 1. Klik Build -> Android App (.apk).
  2. 2. Tunggu 100%.
  3. 3. Scan QR / Download.
